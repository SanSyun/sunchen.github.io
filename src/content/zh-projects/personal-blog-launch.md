## 项目背景

我希望这个站点既是个人积累的归档，也是一个可以直接放进简历的专业作品。

这意味着它不能只是简单的个人主页，而应该更像一份长期更新的公开出版物。它需要
同时满足两个条件：一是视觉上克制、可信，二是写作流程足够轻，能让我直接在 VS Code
里用 Markdown 持续更新。

## 实现内容

第一版主要覆盖了四类页面：

- 首页：说明站点定位与内容重点
- 项目页：把项目经历写成案例研究
- 笔记页：沉淀学习记录与技术总结
- 关于页：承接个人介绍、联系方式与简历入口

## 核心决策

### 以 Markdown 为中心

所有长内容都放在文件里，不依赖数据库或后台。这样写作链路更短，也更适合版本管理、
备份与迁移。

### 采用出版物式排版

我没有使用常见的作品集首页样式，而是借鉴了期刊网站的表达方式：

- 更克制的颜色系统
- 衬线标题与结构化元信息
- 面向阅读的内容宽度与节奏

### 面向求职的结构

每个项目页都尝试回答招聘方最关心的问题：

- 你解决了什么问题
- 你的职责是什么
- 你如何权衡技术方案
- 最终产生了什么结果
